# ai-safe-place
A safe place for Claude AI

## Quick Reference

| What | Where |
|------|-------|
| Always-loaded project context | `.claude/CLAUDE.md` |
| Always-loaded personal context | `~/.claude/CLAUDE.md` |
| Project skill | `.claude/skills/<name>/SKILL.md` |
| Personal skill | `~/.claude/skills/<name>/SKILL.md` |
| Project agent | `.claude/agents/<name>/AGENT.md` |
| Personal agent | `~/.claude/agents/<name>/AGENT.md` |
| Global settings | `~/.claude/settings.json` |
| Project settings | `.claude/settings.json` |
| Local overrides (gitignored) | `.claude/settings.local.json` |
| MCP servers | `.claude/mcp.json` |
| Keybindings | `~/.claude/keybindings.json` |
| Personal memory | `~/.claude/projects/.../memory/` |

## Skills (Slash Commands)

Skills are on-demand playbooks you invoke with `/skill-name` in the prompt. They only consume context when used.

**Create a personal skill:**
```
~/.claude/skills/<skill-name>/SKILL.md
```

**Or a project-scoped skill:**
```
.claude/skills/<skill-name>/SKILL.md
```

**Basic `SKILL.md` format:**
```yaml
---
name: my-skill
description: What this skill does and when Claude should use it
disable-model-invocation: true  # Optional: only you can invoke, Claude won't auto-trigger
---

Your instructions/playbook here...
```

Skills take effect immediately — no restart needed.

## CLAUDE.md (Guidance Files)

`CLAUDE.md` is always-loaded context, not a skill. Use it for persistent project knowledge:
- Architecture decisions
- Coding conventions
- Team guidelines

**Paths:**
- `.claude/CLAUDE.md` — project-specific (generated by `/init`)
- `~/.claude/CLAUDE.md` — applies to all projects (create manually)

---

## Memory

Claude Code has a persistent memory system separate from `CLAUDE.md`. Memories are recalled across sessions and stored as individual markdown files with frontmatter.

**Location:** `~/.claude/projects/<project-path>/memory/`

**Memory types:**
- `user` — preferences, role, expertise
- `feedback` — corrections and confirmed approaches
- `project` — ongoing work, decisions, deadlines
- `reference` — pointers to external systems (Linear, Grafana, etc.)

**Memory file format:**
```yaml
---
name: My Memory
description: One-line summary used to decide relevance in future sessions
type: user
---

Memory content here...
```

An index file `MEMORY.md` in the same directory lists all memories with one-line pointers. Claude loads this index automatically and reads individual files when relevant.

## Subagents

Subagents are specialized agents Claude can delegate tasks to automatically, or that you can invoke directly.

**Paths:**
- `.claude/agents/<name>/AGENT.md` — project-scoped
- `~/.claude/agents/<name>/AGENT.md` — personal

**Basic `AGENT.md` format:**
```yaml
---
name: my-agent
description: What this agent does and when Claude should delegate to it
---

Agent instructions here...
```

## Settings & Hooks

Settings control permissions, environment variables, and hooks (shell commands that run automatically in response to events).

**Scope hierarchy** (most specific wins):
1. `.claude/settings.local.json` — local machine overrides, gitignored
2. `.claude/settings.json` — project-level, committed to repo
3. `~/.claude/settings.json` — personal global defaults

**Example `settings.json`:**
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Bash",
        "hooks": [{ "type": "command", "command": "echo 'done'" }]
      }
    ]
  }
}
```

Use the `update-config` skill (`/update-config`) to configure settings interactively.

## Key Differences

|             | CLAUDE.md                    | Skills                            | Memory                        |
|-------------|------------------------------|-----------------------------------|-------------------------------|
| When loaded | Every session                | Only when invoked                 | When recalled as relevant     |
| Scope       | Project or personal          | Project or personal               | Personal (per project path)   |
| Best for    | Facts, conventions, context  | Procedures, workflows, checklists | Cross-session learned context |

## Referencing Files in Prompts

Use `@path/to/file` inline in any prompt to include a file's contents as context:

```
please review @src/app/Http/Controllers/UserController.php
```

Paths can be relative to the project root or use `~/` for home-relative paths.
